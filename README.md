# Летняя практика: LLM, Prompt Engineering и RAG

**Цель:** изучить работу LLM, эмбеддинги, построить простой Retrieval-Augmented Generation (RAG) проект.  
**Инструменты:** Яндекс LLM, Hugging Face, SentenceTransformers, LangChain, FAISS, Git.

---

## План практики

### 1. Понимание эмбеддингов и механизма self-attention
**Задание:**
- Изучить, что такое эмбеддинги текста и зачем они нужны. (https://habr.com/ru/articles/947216/)
- Понять принцип self-attention в трансформерах. (https://dtf.ru/id2687299/3861067-vnimanie-v-ii-kak-rabotayet-i-pochemu-vazhno?clckid=bc8adce8)
---

### 2. Изучение Hugging Face
**Задание:**
- Ознакомиться с платформой Hugging Face.(https://huggingface.co/)

**Дополнительно на будущее курс по LLM(псоле практики на самостоятельное изучение):**
https://huggingface.co/learn/llm-course/chapter1/1

---

### 3. Развертывание локальной модели эмбеддингов
**Задание:**
- Использовать модель `sentence-transformers/all-MiniLM-L6-v2` с помощью библиотеки Python sentence_transformers.
- Получить эмбеддинги для текста:
  - `"Привет, Мир!"`
  - `"Привет, ПММ!"`
- Рассчитать косинусное сходство между эмбеддингами.

**Ожидаемый результат:**
- Скрипт с результатами эмбеддингов и небольшое сравнение по косинусной мере.

---

### 4. Изучение основ LangChain и text_splitter
**Задание:**
- Ознакомиться с LangChain.(https://tproger.ru/articles/chto-takoe-langchain---frejmvork-python-dlya-ii)
- Ознакомиться с `text_splitter`(https://langchain-doc.readthedocs.io/en/latest/modules/indexes/examples/textsplitter.html)
- Выбрать текст
- Попробовать разбиение текста на чанки с помощью `text_splitter`. 

**Ожидаемый результат:**
- Код, который разбивает текст на логические части.

---

### 5. Изучение векторных баз данных и retriever
**Задание:**
- Прочитать про FAISS и концепцию retriever. (https://habr.com/ru/companies/okkamgroup/articles/509204/)
- Взять код из примера протестировать поиск похожих текстов.

**Ожидаемый результат:**
- Рабочий поиск по эмбеддингам, пример поиска похожих фраз.

---

### 6. Разбор базового примера RAG
**Задание:**
- Разобрать пример baseline RAG:
  - загрузка текста
  - разбиение на чанки
  - построение эмбеддингов
  - поиск
  - генерация ответа

**Ожидаемый результат:**
- Понимание пайплайна RAG и функций каждого шага.

---

### 7. Создание собственного RAG на основе выбранного документа
**Задание:**
- Выбрать текстовый документ.
- Изменить инструкции и параметры.
- Протестировать RAG на своем документе.

**Ожидаемый результат:**
- Рабочий RAG, который отвечает на вопросы по выбранному тексту.

---

### 8. Работа в Git для отслеживания прогресса
**Задание:**
- Вести репозиторий с кодом и отчетами.
- Делать регулярные коммиты с комментариями.

**Ожидаемый результат:**
- Полноценный репозиторий с историей выполнения всех заданий, готовый к презентации.
